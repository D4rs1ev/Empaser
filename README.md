                             1. ОПИСАНИЕ ПРОГРАММЫ
════════════════════════════════════════════════════════════════════════════════════════════════

Email Parser (Empaser) - это универсальный инструмент для анализа email сообщений в формате EML.
Программа предназначена для использования в расследованиях, анализе безопасности и
исследовании почтовых данных.

🔹 ОСНОВНЫЕ ВОЗМОЖНОСТИ:

   ✅ Извлечение всех писем из EML файлов (номера 1.eml, 2.eml, ... до 10000)
   ✅ Интеллектуальный поиск - проверка следующих 5 файлов при пропуске
   ✅ Анализ отправителей (имя, email, домен, IP)
   ✅ Проверка аутентификации (SPF, DKIM, DMARC)
   ✅ Извлечение вложений с вычислением хешей (MD5, SHA1, SHA256)
   ✅ Проверка репутации через VirusTotal (домены, IP, хеши)
   ✅ Обнаружение ссылок на облачные хранилища (Google Drive, Dropbox, Mega и др.)
   ✅ Создание подробных HTML и CSV отчетов
   ✅ Интерактивный просмотр базы данных с фильтрацией и пагинацией
   ✅ Сохранение истории анализов (каждый анализ в отдельной папке с датой/временем)

🔹 ПОДДЕРЖИВАЕМЫЕ ОБЛАЧНЫЕ ХРАНИЛИЩА:

   • Google Drive, Dropbox, OneDrive, iCloud, Box, Mega.nz
   • Яндекс.Диск, Mail.ru Cloud, pCloud, Sync.com
   • Файлообменники: RapidGator, TurboBit, NitroFlare, Uploaded, ZippyShare, MediaFire и др.

                             2. УСТАНОВКА
════════════════════════════════════════════════════════════════════════════════════════════════

🔹 ТРЕБОВАНИЯ:

   • Операционная система: Linux (Kali, Ubuntu, Debian, Mint)
   • Python 3.8 или выше
   • SQLite3
   • Интернет-соединение (для онлайн-проверок)

🔹 УСТАНОВКА:

   1. Создайте директорию для программы:
      mkdir ~/empaser
      cd ~/empaser

   2. Поместите следующие файлы в директорию:
      • empaser.sh              - главный скрипт
      • email_analyzer_advanced.py - анализатор писем
      • email_viewer.py         - просмотрщик базы данных

   3. Сделайте скрипт исполняемым:
      chmod +x empaser.sh

   4. Запустите программу:
      ./empaser.sh

   5. В главном меню выберите пункт [1] для установки зависимостей


════════════════════════════════════════════════════════════════════════════════════════════════
                             3. ПОДГОТОВКА ДАННЫХ
════════════════════════════════════════════════════════════════════════════════════════════════

Для работы программы необходимы EML файлы писем.

🔹 ЭКСПОРТ ПИСЕМ:

   • Microsoft Outlook: Файл → Сохранить как → формат .eml
   • Mozilla Thunderbird: ПКМ на письме → Сохранить как → файл .eml
   • Gmail: Открыть письмо → ... → Скачать сообщение
   • Любой почтовый клиент: экспорт в формат EML

🔹 ПОДГОТОВКА ФАЙЛОВ:

   1. Переименуйте файлы в формате: 1.eml, 2.eml, 3.eml, ... (до 10000)
   2. Поместите все файлы в одну директорию
   3. Файлы могут идти не подряд (1, 5, 10, 100) - программа автоматически
      проверит следующие 5 файлов при обнаружении пропуска


════════════════════════════════════════════════════════════════════════════════════════════════
                             4. НАСТРОЙКА API КЛЮЧА
════════════════════════════════════════════════════════════════════════════════════════════════

VirusTotal API ключ необходим для проверки репутации доменов, IP адресов и хешей файлов.

🔹 КАК ПОЛУЧИТЬ API КЛЮЧ:

   1. Перейдите на сайт: https://www.virustotal.com/gui/join-us
   2. Зарегистрируйтесь (бесплатно)
   3. После входа перейдите в настройки профиля
   4. Скопируйте API ключ

🔹 НАСТРОЙКА:

   1. В главном меню выберите пункт [2]
   2. Вставьте API ключ
   3. Ключ будет сохранен в файл config/vt_config.sh

🔹 ПРИМЕЧАНИЕ:

   • Без API ключа доступен только локальный анализ (без проверки репутации)
   • Бесплатный тариф имеет ограничение: 4 запроса в минуту


════════════════════════════════════════════════════════════════════════════════════════════════
                             5. АНАЛИЗ ПИСЕМ
════════════════════════════════════════════════════════════════════════════════════════════════

🔹 ЗАПУСК АНАЛИЗА:

   1. В главном меню выберите пункт [3]
   2. Укажите путь к директории с EML файлами
   3. Выберите режим анализа:
      • [1] Быстрый анализ (только локальный, без проверок)
      • [2] Полный анализ (с проверкой репутации, требуется API ключ)
      • [3] Полный анализ + извлечение вложений

🔹 ВОЗМОЖНЫЕ ПАРАМЕТРЫ:

   Программа автоматически создает папку с результатами в формате:
   results/analysis_YYYYMMDD_HHMMSS/

   Где:
      YYYY - год
      MM   - месяц
      DD   - день
      HH   - час
      MM   - минута
      SS   - секунда

🔹 ПРИМЕР:

   results/analysis_20241215_143022/

                             6. ПРОСМОТР БАЗЫ ДАННЫХ
════════════════════════════════════════════════════════════════════════════════════════════════

🔹 ЗАПУСК ПРОСМОТРЩИКА:

   • Из главного меню выберите пункт [4] и укажите путь к базе данных
   • Или выберите пункт [5] для просмотра списка всех сохраненных результатов

🔹 ГЛАВНОЕ МЕНЮ ПРОСМОТРЩИКА:

   [1] - Поиск по email
   [2] - Поиск по домену
   [3] - Поиск по IP
   [4] - 👥 Все отправители
   [5] - 🌐 Все домены (с репутацией)
   [6] - 📎 Все вложения
   [7] - ☁️ Облачные хранилища
   [8] - 🚨 Засвеченные индикаторы
   [9] - 🔍 Фильтрация писем
   [0] - 📊 Полная статистика
   [q] - Выход

🔹 НАВИГАЦИЯ ПРИ ПРОСМОТРЕ ПИСЬМА:

   [n] или [→]  - следующее письмо
   [p] или [←]  - предыдущее письмо
   [j]          - перейти к письму по номеру
   [s]          - сохранить текущее письмо в файл
   [a]          - сохранить все письма в папку
   [f]          - найти в тексте
   [u]          - открыть ссылку
   [v]          - показать вложения
   [c]          - показать облачные хранилища
   [q]          - выход

                             7. ФОРМАТЫ ВЫХОДНЫХ ДАННЫХ
════════════════════════════════════════════════════════════════════════════════════════════════

После анализа в директории результатов создаются следующие файлы:

🔹 БАЗА ДАННЫХ:

   email_analysis.db - SQLite база данных со всеми данными

🔹 CSV ОТЧЕТЫ:

   attachments.csv          - все вложения с хешами и рейтингом VT
   hashes.csv               - уникальные хеши с рейтингом
   domains_with_reputation.csv - домены с SPF/DKIM/DMARC и VT
   ips_with_reputation.csv  - IP адреса с рейтингом VT
   senders_with_reputation.csv - отправители с рейтингом
   urls.csv                 - все найденные ссылки
   cloud_storage_links.csv  - ссылки на облачные хранилища
   compromised_indicators.csv - засвеченные индикаторы

🔹 HTML ОТЧЕТ:

   report.html - визуальный отчет с группировкой по категориям угроз

🔹 ВЛОЖЕНИЯ:

   attachments/ - папка с извлеченными файлами (если включено)


════════════════════════════════════════════════════════════════════════════════════════════════
                             8. РАСШИФРОВКА СТАТУСОВ
════════════════════════════════════════════════════════════════════════════════════════════════

🔹 РЕПУТАЦИЯ (reputation_status):

   good       - хорошая репутация, можно доверять
   warning    - есть подозрения, рекомендуется проверить
   bad        - плохая репутация, рекомендуется блокировка
   malicious  - вредоносный объект, требуется блокировка
   suspicious - подозрительный объект, требует проверки
   unknown    - нет данных о репутации

🔹 SPF/DKIM/DMARC СТАТУСЫ:

   valid      - запись существует и валидна
   invalid    - запись существует, но невалидна
   not_found  - запись не найдена
   error      - ошибка при проверке
   unknown    - статус не определен

🔹 VIRUSTOTAL СТАТУСЫ:

   clean      - чистый (0 детекций)
   suspicious - подозрительный (1-5 детекций)
   malicious  - вредоносный (>5 детекций)
   not_found  - не найден в базе VT
   error      - ошибка при проверке

                             9. ПРИМЕРЫ ИСПОЛЬЗОВАНИЯ
════════════════════════════════════════════════════════════════════════════════════════════════

🔹 ПОЛНЫЙ ЦИКЛ РАБОТЫ:

   1. Запуск программы:
      ./empaser.sh

   2. Установка зависимостей:
      Выберите [1] → дождитесь завершения

   3. Настройка API ключа (опционально):
      Выберите [2] → введите ключ

   4. Анализ писем:
      Выберите [3] → укажите путь к EML → выберите режим

   5. Просмотр результатов:
      Выберите [4] или [5] → просмотр базы данных

🔹 БЫСТРЫЙ АНАЛИЗ (без API ключа):

   ./empaser.sh → [3] → путь к EML → [1]

🔹 ПОЛНЫЙ АНАЛИЗ (с API ключом):

   ./empaser.sh → [2] → ввод ключа → [3] → путь к EML → [2]

🔹 ПРОСМОТР ПРЕДЫДУЩИХ РЕЗУЛЬТАТОВ:

   ./empaser.sh → [5] → выбрать нужный результат

🔹 ЭКСПОРТ БАЗЫ ДАННЫХ В CSV:

   sqlite3 -header -csv results/analysis_*/email_analysis.db "SELECT * FROM emails;" > export.csv

                             10. ЧАСТЫЕ ОШИБКИ И РЕШЕНИЯ
════════════════════════════════════════════════════════════════════════════════════════════════

🔹 "Виртуальное окружение не найдено"

   Решение: Выберите пункт [1] в главном меню для установки зависимостей

🔹 "ModuleNotFoundError: No module named 'xxx'"

   Решение: Повторите установку пунктом [1]

🔹 "Файл email_analyzer_advanced.py не найден"

   Решение: Поместите все файлы в одну директорию

🔹 "API key invalid" или "Connection timeout"

   Решение: Проверьте интернет-соединение и правильность API ключа

🔹 "Permission denied"

   Решение: Сделайте скрипт исполняемым: chmod +x empaser.sh

🔹 "Директория не найдена"

   Решение: Укажите полный путь к директории с EML файлами
